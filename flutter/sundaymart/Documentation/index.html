Admin panel Introduction

This project is written in laravel for backend, react.js for frontend and mysql for database.

Laravel is a web application framework with expressive, elegant syntax. We’ve already laid the foundation — freeing you to create without sweating the small things.

React (also known as React.js or ReactJS) is an open-source, front-end, JavaScript library for building user interfaces or UI components. It is maintained by Facebook and a community of individual developers and companies. React can be used as a base in the development of single-page or mobile applications

Server requirements

  • Php version 7.4+
  • Composer
  • MySQL 5.7+
  • Nodejs version 14.0+
  • Npm

Upload on virtual server

  • 1. Zip all files except (node_modules folder)
  • 2. Upload the zip file to the server
  • 3. Unzip the uploaded file
  • 4. Run npm install to install node modules.

Create database on virtual server

Follow the steps below to create the database

  • 1. Create mysql database on your server
  • 2. Enter username, password and database name to .env
  • 3. Run php artisan migrate to create tables in database
  • 4. Run php artisan db:seed code to create data in database

Run on virtual server

  • 1. Run npm run prod to create frontend build files.
  • 2. Configure apache to run project

Configuration

Before running the app, you need to set up several configurations

  • 1. Download and extract the project to any folder
  • 2. Open project in preferred IDE
  • 3. Open terminal in project folder and run flutter pub get to install all packages
  • 4. Open /lib/src/config/global_config.dart and replace the URL with the URL of your server
  • 5. Open /lib/src/config/sms_config.dart and add your nexmo api key. Add the secret key if you want to use nexmo for verification.
  • 6. Create your own Google Maps API key
  • 7. Create the project in firebase and get the cloud messaging key.
  • 8. Firebase configuration
    1. Create a new Firebase project if you don’t have one yet. You can follow this guide for the creation process. NOTE: Please don’t forget to put SHA 1 and SHA 256.
    2. Under the project, select authentication, then enable Phone, Facebook and Google.
    3. For the case of Facebook. We need APP_ID and APP_SECRET. We can obtain these from Facebook Developer Console.
    4. Head over to Facebook Developer Console.
    5. Click on My Apps.
    6. Click the Create App button.
    7. Select Business then continue.
    8. Enter your App Name, Contact Email and App Purpose then, continue to create the app.
    9. In the dashboard, click basic under settings. You will be redirected to a page showing the App_ID` and secret as shown below.
    10. Copy the App_id and Secret then, paste to the fields we left empty on the Facebook sign-in method.
    11. Copy the OAuth redirect URI we need to set up our app with Facebook.
    12. In the Facebook console, head over to the dashboard.
    13. Click setup Facebook login.
    14. On the next page, select Android.
    15. In your AndroidManifest file, copy the package name of your app and paste it into the package name field.
    16. For the default activity name, use youp_packagename.MainActivity.
    17. We need to generate a Development Key Hash. Paste the command below in your terminal: 
      keytool -exportcert -alias androiddebugkey -keystore "C:\Users\USERNAME\.android\debug.keystore" | "PATH_TO_OPENSSL_LIBRARY\bin\openssl" sha1 -binary | "PATH_TO_OPENSSL_LIBRARY\bin\openssl" base64

      Note that you should have openssl-for-windows and JDK installed for Windows users. If you use macOS, use the command below:

      keytool -exportcert -alias androiddebugkey -keystore ~/.android/debug.keystore | openssl sha1 -binary | openssl base64
    18. This generates a key hash unique to your development environment. Copy and paste it into the key hashes field. Save then continue.
    19. Navigate to /app/res/values then create a string resource file named strings.xml.Add the snippets below into that file. 
      <?xml version="1.0" encoding="utf-8"?>
<resources>
    <string name="app_name">Social Auth</string>
    <string name="facebook_app_id">YOUR_APP_ID</string>
    <string name="fb_login_protocol_scheme">fbYOUR_APP_ID</string>
</resources>
      Next, head to /app/manifest/AndroidManifest.xml then enable internet permission by adding the snippets below after the application element. 
      <meta-data android:name="com.facebook.sdk.ApplicationId"
    android:value="@string/facebook_app_id"/>
<activity android:name="com.facebook.FacebookActivity"
    android:configChanges=
        "keyboard|keyboardHidden|screenLayout|screenSize|orientation"
    android:label="@string/app_name" />
<activity
    android:name="com.facebook.CustomTabActivity"
    android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <category android:name="android.intent.category.BROWSABLE" />
        <data android:scheme="@string/fb_login_protocol_scheme" />
    </intent-filter>
</activity>
    20. In the developer’s console dashboard, go to App review.
    21. Under Permissions and Features request advanced access for the public profile.
    22. Your app should now be able to access a Facebook profile.
    23. Access for the public profile.

Android configuration

  • 1. Create and download your app icon
  • 2. After downloading, replace the icons folder with the following folders:
    1. /mipmap-hdpi in /android/app/src/main/res/ folder
    2. /mipmap-mdpi in /android/app/src/main/res/ folder
    3. /mipmap-xhdpi in /android/app/src/main/res/ folder
    4. /mipmap-xxhdpi in /android/app/src/main/res/ folder
    5. /mipmap-xxxhdpi in /android/app/src/main/res/ folder
  • 3. Open /android/app/build.gradle and change the package name
  • 4. Open all AndroidManifest.xml files from /android/app/src/main/, /android/app/src/profil/, /android/app/src/debug/ folders and set package name and application name.
  • 5. Open /android/app/src/main/kotlin/-- Package name --/MainActivity.kt and change the package name
  • 6. Follow Flutter google map documentationto configure app for android
  • 7. Follow Push notification documentation to configure push notification for android

Ios configuration

  • 1. Create icons with following sizes
    1. 40x40
    2. 58x58
    3. 60x60
    4. 80x80
    5. 87x87
    6. 120x120
    7. 180x180
  • 2. Click Assets.xcassets in the Project navigator and choose AppIcon. This will give you an empty app icon set. 
    Drag the right-sized image (in .png format) from Finder onto every blank in the app settings. The app icon should be all set up by this moment. 
  • 3. Follow Flutter google map documentation to configure app for ios
  • 4. Follow Push notification documentation to configure push notification for ios

Run & Build

If you want to run the app on an emulator, run flutter run
If you want to build apk, follow the steps below:

  • To create apk file, run flutter build apk --release
  • To generate an app bundle, run flutter build appbundle --target-platform android-arm,android-arm64,android-x64
  • To split the APKs per ABI, run flutter build apk --target-platform android-arm,android-arm64,android-x64 --split-per-abi
  • To install your application on your connected device run flutter install
  • If you want to upload your application to Google Play you must sign it before uploading by generating a signing key by running the following command:
    keytool -genkey -v -keystore android/app/key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias key Open /android/key.properties and edit the following attributes:
  • After editing, re-build your application
